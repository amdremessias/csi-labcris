Guia de Deploy: OpenClaude & Ollama (Llama)

Este guia detalha a configuração de um ambiente isolado em containers Docker no Zorin OS 18 para rodar a interface OpenClaude utilizando o modelo Llama localmente.

1. Estrutura de Diretórios

Organize o seu laboratório criando uma pasta dedicada no seu SSD:

mkdir -p ~/Projetos/ia-local
cd ~/Projetos/ia-local
mkdir workdir

2. Configuração do Docker Compose

Crie o arquivo docker-compose.yml com a definição dos serviços:

version: '3.8'

services:
  ollama:
    image: ollama/ollama:latest
    container_name: ollama-service
    volumes:
      - ./ollama_data:/root/.ollama
    ports:
      - "11434:11434"
    restart: unless-stopped

  openclaude:
    build: 
      context: https://gitlawb.com/node/repos/z6MkqDnb/openclaude.git
    container_name: openclaude-app
    depends_on:
      - ollama
    stdin_open: true
    tty: true
    environment:
      - CLAUDE_CODE_USE_OPENAI=1
      - OPENAI_API_KEY=ollama
      - OPENAI_BASE_URL=http://ollama:11434/v1
      - OPENAI_MODEL=llama3
    volumes:
      - ./workdir:/app/workdir
    restart: unless-stopped

3. Execução e Instalação do Modelo

Suba os containers e baixe o modelo Llama 3:

# Iniciar containers
docker compose up -d

# Baixar o modelo Llama 3 (Open Source)
docker exec -it ollama-service ollama run llama3
Dica: Uma vez que o download terminar, digite /exit para fechar o prompt do Ollama sem desligar o serviço.

4. Acesso ao OpenClaude

Para interagir com o OpenClaude dentro do ambiente isolado:

docker exec -it openclaude-app sh

5. Dicas de Otimização

  • Persistência: Os modelos ficam salvos na pasta ollama_data, garantindo que não se percam após reiniciar.
  • Privacidade: Todo o processamento ocorre no seu hardware, sem envio de dados para nuvem.
  • Monitoramento: Use docker stats para acompanhar o consumo de RAM no seu Zorin OS.
