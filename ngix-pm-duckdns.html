Configuração de HTTPS e Proxy Reverso

Guia técnico para implementação do Nginx Proxy Manager com DuckDNS no sistema Ticketz.

1 Objetivo do Projeto

O foco desta implementação foi resolver a indisponibilidade das APIs de mídia (microfone) causadas pela falta de ambiente seguro (HTTPS). Utilizamos o Nginx Proxy Manager (NPM) para gerenciar certificados e tráfego via DuckDNS.

2 Infraestrutura de Rede

Criamos uma rede bridge externa para integrar os containers do sistema ao Proxy:

docker network create nginx-proxy

No Docker Compose, os serviços frontend e backend foram conectados a esta rede.

3 Ajuste das Variáveis (.env)

Para evitar erros de Mixed Content, as URLs foram atualizadas para HTTPS:

FRONTEND_HOST=novochat.duckdns.org

BACKEND_PROTOCOL=https

REACT_APP_BACKEND_URL=https://novochat.duckdns.org/backend

4 Nginx Proxy Manager

  • SSL: Let's Encrypt via DNS Challenge com DuckDNS Token.
  • Websockets Support: Ativado para funcionamento das notificações em tempo real.
  • Force SSL: Ativado para garantir que todo acesso seja via porta 443.
