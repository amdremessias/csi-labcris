Configurando Nginx Proxy Manager com DuckDNS e SSL

Este guia detalha a configuração do Nginx Proxy Manager (NPM) para resolver problemas de acesso a recursos como microfone no serviço Ticketz.

1. Obtenha seu Token do DuckDNS

O token é essencial para que o NPM valide a propriedade do domínio via DNS Challenge:

  • Acesse o site DuckDNS.
  • Copie seu Token (ex: 26887e1b-e75c-43d8-9d16-5f11135bb84d).

2. Configuração da Rede Docker

Ambos os serviços precisam estar na mesma rede para comunicação interna segura:

docker network create nginx-proxy

3. Ajuste no Docker Compose do Ticketz

Edite o docker-compose.yaml do serviço frontend:

frontend:
  expose:
    - 80
  networks:
    - ticketz
    - nginx-proxy
Nota: Remova a seção ports para evitar exposição direta e forçar o tráfego pelo Proxy Reverso.

4. Configuração do SSL via DNS Challenge

No painel do NPM (porta 81), ao adicionar o Proxy Host para novochat.duckdns.org:

  • Selecione Request a new SSL Certificate.
  • Ative Use a DNS Challenge e escolha DuckDNS.
  • Insira a configuração: dns_duckdns_token = seu_token_aqui.
  • Aumente o Propagation Seconds para 120 para garantir a sincronia.

5. Variáveis de Ambiente (.env)

Atualize os arquivos .env-frontend e .env-backend para usar HTTPS e evitar erros de "Mixed Content":

FRONTEND_HOST=novochat.duckdns.org
REACT_APP_BACKEND_URL=https://novochat.duckdns.org/backend
BACKEND_PROTOCOL=https
