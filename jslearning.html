Por que essa aula importa

Você não consegue escrever uma linha de JavaScript útil sem dominar variáveis e tipos. É o alfabeto. E o console é seu microscópio — sem ele, você programa no escuro, chutando.

Essa aula é o "absoluto zero". Vamos do console.log aos operadores modernos (??, ?., spread). No fim, você consegue ler 70% do JavaScript que aparece em qualquer projeto.

O console: sua melhor ferramenta

Abra qualquer site, aperte F12, vai na aba "Console". Pronto: você tem um REPL JavaScript rodando ali. Tudo que você digitar executa.

console.log("Olá, mundo!")
console.log("A soma é:", 2 + 3)
console.log({ nome: "Ana", idade: 30 })
console.error("Algo deu errado")
console.warn("Cuidado com isso")
console.table([{ nome: "Ana" }, { nome: "Bia" }])
  • console.log é o pão de cada dia.
  • console.error aparece vermelho com ícone de erro.
  • console.warn aparece amarelo.
  • console.table renderiza arrays de objetos como tabela — lindo pra debug.

Quem domina o console economiza horas vs quem tenta "adivinhar" o que tem dentro de uma variável.

Variáveis: let, const, var

Três formas de declarar. Em JS moderno, você usa const por padrão e let quando precisa reatribuir. var virou peça de museu.

const PI = 3.14159         // não muda mais — use sempre que possível
let contador = 0           // vai mudar ao longo do código
contador = contador + 1    // ok, foi declarado com let

// PI = 4    // ❌ TypeError: Assignment to constant

Diferenças cruciais:

  • const e let têm escopo de bloco ({}). var tem escopo de função (confuso, evite).
  • const impede reatribuição, mas se for objeto/array, o conteúdo ainda pode mudar:
    const lista = [1, 2, 3]
lista.push(4)   // ok, lista agora é [1,2,3,4]
// lista = [99] // ❌ não pode reatribuir a referência

Tipos primitivos

JavaScript tem 7 tipos primitivos:

const texto = "uma string"     // string
const tambemTexto = 'aspas simples também' // string
const numero = 42               // number (int e float são o mesmo tipo)
const flutuante = 3.14          // number
const verdadeiro = true         // boolean
const falso = false             // boolean
const nada = null               // null (vazio intencional)
const indef = undefined         // undefined (não atribuído)
const gigante = 9007199254740993n // bigint (raro)
const sym = Symbol("id")         // symbol (raro)

null vs undefined: null é "explicitamente nada" (você setou). undefined é "nunca foi setado". Na prática, trate igual.

Para descobrir o tipo:

typeof "abc"      // "string"
typeof 42         // "number"
typeof true       // "boolean"
typeof undefined  // "undefined"
typeof null       // "object" 😅 (bug histórico, conviva)
typeof [1,2]      // "object"
typeof {a:1}      // "object"

Strings: o básico essencial

const nome = "Luis"
const sobrenome = "Martins"

// Concatenação clássica (evite)
const fullName = nome + " " + sobrenome

// Template literal (use sempre) — usa crases (backtick)
const saudacao = `Olá, ${nome} ${sobrenome}!`
const multiLinha = `
  Linha 1
  Linha 2
`

// Métodos úteis
"abc".length             // 3
"abc".toUpperCase()      // "ABC"
"  oi  ".trim()          // "oi"
"olá".startsWith("o")    // true
"olá".includes("á")      // true
"a,b,c".split(",")       // ["a","b","c"]
["a","b","c"].join("-")  // "a-b-c"

Template literal com ${} é o jeito moderno. Use sempre.

Números

2 + 3          // 5
10 / 3         // 3.333333...
10 % 3         // 1 (resto)
2 ** 10        // 1024 (potência)

Math.floor(3.7)    // 3
Math.ceil(3.2)     // 4
Math.round(3.5)    // 4
Math.random()      // 0 a 0.99...
Math.max(1,5,3)    // 5

Number("42")       // 42 (converte string)
parseInt("42px")   // 42 (lê até onde dá)
parseFloat("3.14")

Cuidado: JavaScript não tem tipo "inteiro" separado de "float". 0.1 + 0.2 dá 0.30000000000000004. Coisas de ponto flutuante (existe em qualquer linguagem). Pra dinheiro, trabalhe em centavos (inteiros), nunca em reais com vírgula.

Booleanos e "truthy"/"falsy"

JavaScript trata como false em contextos booleanos:

  • false
  • 0
  • "" (string vazia)
  • null
  • undefined
  • NaN

Qualquer outra coisa é true ("truthy"). Inclusive "0" (string com zero), [], {} — todos truthy.

if (usuario) {  // truthy/falsy implícito
  console.log("Tem usuário")
}

Operadores essenciais

Comparação

5 === 5     // true (igualdade estrita, USE SEMPRE)
5 === "5"   // false (tipos diferentes)
5 == "5"    // true (igualdade frouxa, EVITE)
5 !== "5"   // true

Regra de ouro: use === e !==. O == faz conversões implícitas que dão bug.

Lógicos

true && false   // false (AND)
true || false   // true (OR)
!true           // false (NOT)

// Curto-circuito útil
const nome = usuario.nome || "Anônimo"   // se nome for falsy, usa "Anônimo"
const idade = usuario.idade ?? 0          // ?? é nullish — só null/undefined viram fallback

Diferença sutil: || cai pra fallback em qualquer falsy (inclui 0 e ""). ?? cai só em null/undefined. Pra valor numérico, use ??:

const tickets = 0
const exibido1 = tickets || "—"  // "—" (errado, 0 é falsy)
const exibido2 = tickets ?? "—"  // 0 (correto)

Ternário

const status = idade >= 18 ? "maior" : "menor"

Spread / rest

const arr1 = [1, 2, 3]
const arr2 = [...arr1, 4, 5]        // [1,2,3,4,5]

const pessoa = { nome: "Ana" }
const comIdade = { ...pessoa, idade: 30 }  // { nome:"Ana", idade:30 }

Arrays e objetos: introdução

const frutas = ["maçã", "banana", "pera"]
frutas[0]              // "maçã"
frutas.length          // 3
frutas.push("uva")     // adiciona no fim

const usuario = {
  nome: "Luis",
  idade: 30,
  ativo: true
}
usuario.nome           // "Luis"
usuario["nome"]        // "Luis"
usuario.idade = 31     // atualiza

Mergulho completo em arrays/objetos vem na aula 3.

Erros comuns

  • Usar var. Use const por padrão, let quando precisar.
  • Comparar com ==. Sempre ===.
  • Esquecer = na atribuição (if (x = 5) é bug — atribui em vez de comparar).
  • null vs undefined confusão. Pra checar "tá vazio", use value == null (checa os dois) ou explicitamente value === null || value === undefined.
  • 0.1 + 0.2 !== 0.3 — bug clássico. Para dinheiro, use centavos.
  • Tentar mudar const que aponta pra objeto. A referência é imutável, mas o conteúdo do objeto é mutável.

Cheatsheet

Operador Significado
=== / !== igualdade estrita
&& / || E / OU
?? fallback se null/undefined
?. optional chaining (user?.nome)
... spread/rest
`...${var}...` template literal

Exercícios

🧪 Exercício 1

Declare uma constante chamada nome com seu nome e use template literal para imprimir "Olá, [nome], bem-vindo!".

Ver resposta
const nome = "Luis"
console.log(`Olá, ${nome}, bem-vindo!`)

🧪 Exercício 2

Crie uma variável idade que será 25 e use ternário para imprimir "maior de idade" ou "menor de idade".

Ver resposta
const idade = 25
console.log(idade >= 18 ? "maior de idade" : "menor de idade")

🧪 Exercício 3

Use ?? para criar uma variável quantidade que recebe o valor 0, e depois imprimi-la com fallback "(vazio)". Note que com 0 o || daria o fallback errado.

Ver resposta
const quantidade = 0
console.log(quantidade ?? "(vazio)") // 0
console.log(quantidade || "(vazio)") // "(vazio)" — ERRADO

🧪 Exercício 4

Use spread para combinar dois arrays [1,2,3] e [4,5,6] num só.

Ver resposta
const a = [1,2,3]
const b = [4,5,6]
const tudo = [...a, ...b]
console.log(tudo) // [1,2,3,4,5,6]

🧪 Exercício 5

Use console.table para visualizar um array de 3 objetos com nome e idade.

Ver resposta
const pessoas = [
  { nome: "Ana", idade: 28 },
  { nome: "Bruno", idade: 34 },
  { nome: "Clara", idade: 22 }
]
console.table(pessoas)
